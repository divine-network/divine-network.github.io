DIVINE NETWORK
A Layer 2 EVM Rollup built on top of Algorand
DIVINE NETWORK introduces
A layer 2 rollup network built on top of Agrorand
Built on Algorand, as a layer 2 network, specifically a "rollup", Divine Network moves state data and computation off-chain, while keeping some data on the layer 1 for security. It inherits all of Algorand's security and decentralization, while still being blazingly fast and scalable
EVM integration
Divine Network has the EVM as the core contract execution environment. It is fully compatible with the original EVM that is from Ethereum and in fact is based on an EVM implementation of an existing Ethereum client, which works perfectly well with our rollup model. The transaction format, the address format, signatures, etc are also the same as the original Ethereum.
Proofless rollup
Prooflessness is the most unique design of Divine Network compared to other rollups. The only reason that many rollups like Arbitrum or zksync require optimistic fraud proof or zk proof is because they are reliant on the layer 1's native token, which introduces the act of bridging the token between the layer 1 and the layer 2. Divine Network has its native token so Divine Network do not need proof, making Divine Network more efficient, easier to upgrade, and simpler to implement decentralized sequencers
Decentralized sequencers
Divine Network's sequencer is fully decentralized, anyone can become a sequencer in the network if they wanted to. There is no stake needed, no governance token needed whatsoever, you can run a sequencer node right from your personal computer, just be sure to have some Algos to pay for gas when submitting rollup batches. This is possible thanks to our proof-less model.
What DIVINE NETWORK brings
Blazing fast transactions and low gas fees, scaling Algorand
Integrating EVM where EVM based dapps can fully function
Leveraging the decentralization of Algorand with the decentralized sequencers system
